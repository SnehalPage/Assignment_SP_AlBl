The Grid

Albelli's interfaces follow a grid-based design. This makes sure all (UI) designers use the same alignment when creating interfaces.

Description

Albelli's user interfaces use a grid-based design, consisting of 12 columns. This keeps relative sizes of components equal in all browser sizes. Grids therefore make our website's interfaces scalable and it is easier to design for. It also gives designers on the same page when it comes to aligning your designs, as we're all using the same grids.

Code, options, actual samples, references

Working with a Grid allows you to create multi-device layouts quickly and easily.
Using a 12-column nestable grid system you can create from rather simple to pretty complex layouts.
To work with different device widths, you have access to two separate grids (small and large). The small grid should be used up to the 768px breakpoint.
If you know that your grid structure will be the same for small devices as it will be on large devices, just use the small grid.

Offset

Offsets allow you to create additional space between columns in a row. Like the rest of the grid, they're nestable. You can use classes like .large-offset-1 and .small-offset-3 to manipulate your column positions in different ways. You can offset up to 11 since you wouldn't ever offset a full-width column.

Centered Columns

Centered columns are placed in the middle of the row. This does not center their content, but centers the grid element, itself. This is accomplished by adding a class of large-centered or small-centered depending on which breakpoint you want the columns to center on. Small versions will carry through all breakpoints if not overridden by a large version. You can center any number of columns you have. To have a column not carry it's small-centered to large, add a class of large-uncentered.

Block Grid

Block grids give you a way to evenly split contents of a list within the grid. If you wanted to create a row of 5 images or paragraphs that need to stay evenly spaced no matter the screen size, the block grid is for you.

Block grids are made from a ul.small-block-grid-#, ul.medium-block-grid-# or ul.large-block-grid-#. These are ideal for blocked-in content generated by an application, as they do not require rows or even numbers of elements to display correctly.

These have a bit of flexibility since you have access to two separate grids between our built in 768px breakpoint. If you use the small-block-grid only, the grid will keep its spacing and configuration no matter the screen size. If you use large-block-grid only, the list items will stack on top of each other for small devices. If you use all small, medium and large classes combined, you can control the configuration and layout separately for each breakpoint.

If you add to the ul a .block-grid-rounded class, the images within the li elements will get rounded corners, a shadow and a css transition hover effect.

<!-- Using only the small-block-grid -->
<ul class="small-block-grid-2">
  <li><img src="cdn/assets/demo/demo1.png"></li>
  <li><img src="cdn/assets/demo/demo2.png"></li>
  <li><img src="cdn/assets/demo/demo3.png"></li>
